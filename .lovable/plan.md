

# منظومة إدارة الصيدلية - المرحلة الأولى (الأساس)

## التصميم العام
- **الألوان**: أخضر صيدلاني (#059669 رئيسي، #047857 داكن، #D1FAE5 فاتح) مع خلفية بيضاء نظيفة
- **الاتجاه**: RTL بالكامل مع خط عربي احترافي
- **التنقل**: شريط جانبي قابل للطي مع قائمة F12 للتنقل السريع

## 1. قاعدة البيانات (Supabase)
إنشاء جميع الجداول الأساسية كما في الوثيقة:
- **Products** (الأصناف) - مع الباركود والتصنيف والأسعار
- **Inventory** (المخزون) - الكميات والدفعات وتواريخ الصلاحية
- **Sales + Sale_Items** (المبيعات وتفاصيلها)
- **Purchases + Purchase_Items** (المشتريات وتفاصيلها)
- **Customers** (الزبائن) مع أرصدة الديون الآجلة
- **Suppliers** (الموردين) مع أرصدة المدفوعات
- **Users** (المستخدمين) مع المصادقة
- **Roles + Role_Permissions** (الأدوار والصلاحيات) - مدير، صيدلي، محاسب، مندوب توصيل
- **Shifts** (الورديات) - فتح وإغلاق مع العهدة النقدية والإلكترونية
- **Expenses** (المصاريف التشغيلية)
- **Controlled_Drugs_Log** (سجل الأدوية المخدرة)
- **Delivery_Orders** (طلبات التوصيل)
- سياسات أمان RLS لكل جدول حسب صلاحيات المستخدم

## 2. المصادقة والصلاحيات
- تسجيل دخول بالبريد وكلمة المرور
- نظام أدوار: مدير، صيدلي، محاسب، مندوب توصيل
- حماية الصفحات حسب صلاحية المستخدم
- صفحة تسجيل الدخول بتصميم احترافي

## 3. شاشة المبيعات السريعة (الشاشة الرئيسية)
- خلية إدخال الباركود مع دعم البحث بالاسم (F2)
- جدول الفاتورة: رقم، اسم الصنف، الاسم العلمي، الصلاحية، السعر، الكمية المتاحة
- تعدد الفواتير (F4) - إمكانية تعليق فاتورة والعودة لها
- خيارات الدفع: نقدي، خدمات (موبي كاش، ادفع لي، يسر، إلخ)، بيع آجل، إتلاف، مفقود
- عرض إجمالي الفاتورة مع زر الدفع
- منطقة مراجعة فواتير اليوم الحالي
- تنبيهات الصلاحية ونفاذ الكمية أثناء البيع

## 4. إدارة المخزون وبطاقة الصنف
- جدول المخزون: الاسم، الباركود، الكمية، أقرب صلاحية، سعر البيع، المورد
- فلترة متقدمة: بالاسم، الباركود، التصنيف، المورد، قرب انتهاء الصلاحية، نفاذ الكمية
- بطاقة الصنف: تعريف صنف جديد وتعديل الموجود
- توليد باركود تلقائي (4 أرقام) إذا لم يوجد
- تتبع الدفعات وتواريخ الصلاحية
- تسجيل الأصناف التالفة والمفقودة

## 5. شاشة المشتريات
- إدخال الباركود/البحث عن الصنف
- تنبيه عند عدم وجود الصنف مع خيار إنشاء بطاقة صنف جديد
- اختيار المورد ورقم الفاتورة مع إمكانية إضافة مورد جديد
- إدخال الكمية وسعر الشراء وسعر البيع والصلاحية
- لوحة معلومات الصنف (الكمية المتاحة وآخر سعر شراء)
- تتبع المدفوعات (مدفوعة/آجلة) مع تنبيهات الاستحقاق

## 6. حسابات الزبائن والموردين والمصاريف
- **الزبائن**: بيانات الزبون، كشف حساب الديون الآجلة، تنبيهات التأخر
- **الموردين**: بيانات المورد، كشف المدفوعات، الرصيد المستحق
- **المصاريف**: واجهة إدخال المصاريف مع تصنيفات فرعية (إيجار، رواتب، كهرباء، إلخ)

## 7. إدارة الورديات
- فتح وردية جديدة مع إدخال العهدة النقدية الأولية
- إغلاق الوردية مع تسليم العهدة النقدية والإلكترونية
- تقرير إقفال الوردية

## 8. لوحة التحكم (للمدير)
- نظرة عامة: ملخص المبيعات، الأرباح، تنبيهات المخزون
- وصول مباشر لجميع أقسام الإدارة
- إدارة الموظفين والصلاحيات والرواتب
- إدارة خدمة التوصيل
- الإعدادات العامة (الفواتير، طرق الدفع، التنبيهات)

## 9. التقارير
- تقارير المبيعات (يومية/شهرية/سنوية، بحسب الصنف/الصيدلي/طريقة الدفع)
- تقارير المخزون (الحالي، الحركة، قرب الانتهاء، التالف/المفقود)
- تقارير المشتريات (بحسب المورد/الصنف)
- تقارير الحسابات (المصاريف، كشوف الزبائن والموردين)
- تقارير الورديات
- إمكانية التصدير إلى PDF/Excel

## 10. خدمة التوصيل
- إدارة طلبات التوصيل واختيار المندوب
- تتبع قيمة الطلبية في حساب المندوب
- تأكيد استلام القيمة وإدخالها في الخزينة

## ملاحظة
هذه المرحلة الأولى تبني النظام كاملاً بجميع وحداته الأساسية. سيتم بناؤه على عدة خطوات متتالية نظراً لضخامته. النظام مصمم ليكون قابلاً للتطوير لاحقاً للعمل أوفلاين (PWA/Service Workers).
